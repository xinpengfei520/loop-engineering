# Safety Policy

> Companion to [`loop-constraints.md`](../loop-constraints.md). Constraints file = the
> rules the loop reads each run; this file = the *why* and the operational policy behind
> them. If the two ever disagree, **the stricter one wins** and the loop must escalate.
>
> Defaults below are **conservative on purpose**. Loosen them deliberately, one line at a
> time, only after the corresponding checklist item in
> [`loop-design-checklist.md`](loop-design-checklist.md) is signed off.

## 1. Denylist ŌĆö paths the loop must never modify

| Pattern | Reason |
|---------|--------|
| `.env`, `.env.*` (except `.env.example`) | Secrets |
| `auth/`, `credentials/`, `secrets/` | Auth material |
| `payments/` | Money-moving code |
| Infra configs (CI/CD, IaC, Dockerfiles, k8s) | Blast radius ŌĆö human approval required |
| `.git/`, lockfiles regenerated by hand | Integrity |

Reading these for context is allowed; **writing/deleting is not**. A match ŌåÆ stop and escalate.

## 2. Auto-merge policy

- **No auto-merge to `main`. Ever.** A human clicks merge.
- The loop may, at L2+, open a **draft PR** with a proposed minimal fix. It stays draft until a human reviews and marks it ready.
- The loop never marks its own PR ready, never dismisses reviews, never force-pushes to a shared branch.

## 3. Write scope by level

| Level | The loop may write toŌĆ” |
|-------|------------------------|
| **L1 (now)** | `STATE.md`, `loop-run-log.md` only. Report-only ŌĆö no code changes. May `git commit` those two bookkeeping files at end of run (never `push`); any other staged path ŌåÆ do not commit, escalate. |
| **L2** | Above + source files **outside the denylist**, one fix per run, in an isolated worktree, behind a draft PR. |
| **L3** | Above + broader scope with explicit per-path gates in `LOOP.md`. |

## 4. MCP / external connector scopes

- Connectors are **read + comment only** until explicitly trusted (see `LOOP.md` ŌåÆ Connectors).
- No connector may write, delete, merge, deploy, or send external messages without a human gate.
- Grant the **narrowest** scope that makes the loop useful; widen per-capability, never wholesale.
- No secret values in prompts, `STATE.md`, or the run log.

## 6. Kill switch

Set **`loop-pause-all`** (see `loop-budget.md`). The `loop-constraints` skill checks it first
each run and exits immediately if active. Resume only after a human clears the flag in `STATE.md`.
